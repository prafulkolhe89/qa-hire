---
name: JobStatus enum pattern
description: JobStatus and similar Orval-generated types are plain TS union types, not runtime enum objects.
---

Types like `JobStatus`, `ListJobsStatus`, `ProfileInputJobTypesItem` generated by Orval are TypeScript union types, not runtime enum objects.

**Why:** Orval generates `export type JobStatus = "new" | "interested" | "applied" | "not_relevant" | "duplicate"` — not `export const JobStatus = { interested: "interested", ... }`. Using `JobStatus.interested` at runtime crashes because there is no runtime object.

**How to apply:** Always use string literals directly: `"interested"`, `"applied"`, `"not_relevant"` etc. Import these types as `import type { JobStatus }` for type annotation only, never to read values from.
